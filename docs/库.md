# ⭐ Mod资产库（Starred Library）设计文档

## 一、背景与目标

本模块用于构建一个**本地优先（Local-first）的Mod资产管理系统**，支持用户对以下资源进行统一管理：

* Mod（模组）
* Modpack（整合包）
* Server（服务器）

系统目标：

1. ✅ 支持离线使用（无网络可浏览、管理）
2. ✅ 支持多设备数据同步（云端仅做数据镜像）
3. ✅ 支持更新检测（由客户端自行完成）
4. ✅ 支持扩展（标签、分组、订阅、分享）

---

## 二、核心设计原则

### 1️⃣ Local-first 架构

* 本地数据为唯一真实来源（Source of Truth）
* 云端不参与业务逻辑，仅做同步存储

---

### 2️⃣ 单表模型 + JSON扩展

* 使用单表 `starred_items` 作为核心数据结构
* 通过 JSON 字段扩展能力（避免频繁改表结构）

---

### 3️⃣ 数据职责分离

```txt
snapshot → 展示数据（离线可用）
state    → 状态数据（更新检测）
meta     → 用户行为（标签/备注）
```

---

### 4️⃣ 去中心化更新机制

* 客户端直接请求第三方 API（Modrinth / CurseForge）
* 本地计算更新状态（hasUpdate）

---

## 三、系统架构

```txt
客户端（核心）
  ├── SQLite（本地数据库）
  ├── 更新检测器（API请求）
  ├── 同步模块（Sync Queue）
  ↓
云端（辅助）
  └── 数据镜像（JSON存储）
```

---

## 四、本地数据库设计

### 4.1 核心表：starred_items

```sql
CREATE TABLE starred_items (
  id TEXT PRIMARY KEY,

  type TEXT NOT NULL,
  source TEXT NOT NULL,
  project_id TEXT,

  title TEXT,
  author TEXT,

  snapshot TEXT NOT NULL,
  state TEXT NOT NULL,
  meta TEXT NOT NULL,

  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);
```

---

### 4.2 系统表：app_meta

```sql
CREATE TABLE app_meta (
  key TEXT PRIMARY KEY,
  value TEXT
);
```

用途：

* device_id
* last_sync_at
* schema_version

---

### 4.3 同步队列表：sync_queue

```sql
CREATE TABLE sync_queue (
  id TEXT PRIMARY KEY,
  action TEXT NOT NULL,
  payload TEXT NOT NULL,
  created_at INTEGER NOT NULL
);
```

用途：

* 离线修改缓存
* 同步重试机制

---

### 4.4 分组表（可选）：collections

```sql
CREATE TABLE collections (
  id TEXT PRIMARY KEY,
  name TEXT,
  created_at INTEGER,
  updated_at INTEGER
);

CREATE TABLE collection_items (
  collection_id TEXT,
  item_id TEXT,
  PRIMARY KEY (collection_id, item_id)
);
```

## 五、数据结构定义

### 5.1 StarredItem

```ts
interface StarredItem {
  id: string;

  type: "mod" | "modpack" | "server";
  source: "modrinth" | "curseforge" | "custom";

  projectId?: string;

  snapshot: {
    title: string;
    iconUrl?: string;
    author?: string;
    description?: string;

    loaders?: string[];
    categories?: string[];

    version?: string;
    updatedAt?: string;
  };

  state: {
    installedVersion?: string;

    lastKnownVersion?: string;
    lastKnownUpdatedAt?: string;

    hasUpdate: boolean;
    lastCheckedAt?: number;
  };

  meta: {
    createdAt: number;
    updatedAt: number;

    tags?: string[];
    note?: string;

    pinned?: boolean;
    archived?: boolean;
  };
}
```

## 六、核心流程设计

### 6.1 添加收藏

```txt
用户点击收藏
  ↓
生成 StarredItem
  ↓
写入 starred_items
  ↓
写入 sync_queue
```

---

### 6.2 更新检测

```txt
打开“我的库”或后台任务
  ↓
遍历 starred_items
  ↓
按 source 调用 API
  ↓
更新 state.lastKnownVersion
  ↓
计算 hasUpdate
  ↓
写回数据库
```

---

### 6.3 云同步

```txt
读取 sync_queue
  ↓
上传云端
  ↓
成功 → 删除队列记录
```

---

### 6.4 冲突解决

规则：

```txt
以 updated_at 为准（Last Write Wins）
```

# 二、表结构设计（最终版）

---

## 2.1 collections（集合表）

```sql
CREATE TABLE collections (
  id TEXT PRIMARY KEY,           -- UUID

  name TEXT NOT NULL,
  description TEXT,

  -- 类型（关键扩展点）
  type TEXT NOT NULL,            -- group / modpack / favorite

  -- 封面 & UI
  cover_image TEXT,

  -- 排序 & 展示
  sort_order INTEGER DEFAULT 0,

  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);
```

---

## 🔥 type 字段说明（非常关键）

```txt
group      → 普通分组（收藏夹）
modpack    → 整合包（核心能力）
favorite   → 系统默认收藏（可选）
```

👉 这样你可以一套结构做多种用途

---

## 2.2 collection_items（集合关系表）

```sql
CREATE TABLE collection_items (
  id TEXT PRIMARY KEY,

  collection_id TEXT NOT NULL,
  item_id TEXT NOT NULL,        -- 对应 starred_items.id

  -- 顺序（用于UI排序）
  position INTEGER DEFAULT 0,

  -- 可扩展（未来）
  extra TEXT,                   -- JSON（例如版本锁定）

  created_at INTEGER NOT NULL,

  UNIQUE (collection_id, item_id)
);
```

---

## 🔥 extra 字段（关键扩展）

用于整合包能力：

```json
{
  "lockedVersion": "1.0.0",
  "required": true
}
```

# 十、关键注意事项（必须看）

---

## ❗ 不要把集合数据写进 starred_items

👉 ❌ 错误：

```json
{
  "collections": [...]
}
```

👉 会导致：

* 数据冗余
* 同步混乱

---

## ❗ 不要允许重复 item

👉 必须：

```sql
UNIQUE (collection_id, item_id)
```

---

## ❗ 不要忽略 position

👉 否则 UI 排序会乱
