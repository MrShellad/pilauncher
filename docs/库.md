# 📦 多资源收藏系统设计文档（Launcher）

## 一、设计目标

构建一个统一的「收藏系统」，支持以下资源类型：

* MOD 收藏
* 服务器收藏
* 整合包收藏（Modpack）
* 资源包收藏（Resource Pack）
* 数据包收藏（Datapack）

并实现：

1. ✅ **跨来源收藏（本地 / CurseForge / Modrinth / 外链）**
2. ✅ **结构化分类与分组管理**
3. ✅ **多维筛选（Loader / MC版本 / 标签）**
4. ✅ **版本兼容感知**
5. ✅ **更新检测与提醒**
6. ✅ **资源关联（Mod ↔ Datapack / ResourcePack）**
7. ✅ **外链统一管理能力**

---

## 二、核心设计思想

### 1️⃣ 收藏 ≠ 下载

收藏系统本质是：

> 👉 “资源索引管理层” 而不是文件系统

也就是说：

* 收藏可以是未下载的
* 可以是外链
* 可以只是一个“引用”

---

### 2️⃣ 所有资源统一抽象

定义统一模型：

```ts
interface FavoriteItem {
  id: string
  type: "mod" | "server" | "modpack" | "resourcepack" | "datapack"

  name: string
  icon?: string

  source: "local" | "curseforge" | "modrinth" | "external"

  externalUrl?: string

  mcVersions: string[]
  loaders?: string[]

  tags?: string[]
  groupIds: string[]

  relatedIds?: string[] // 关联资源

  metadata?: Record<string, any>

  lastCheckedVersion?: string
  updateInfo?: UpdateStatus
}
```

# 二、数据库设计（核心）

## 1️⃣ 收藏主表（favorite_items）

👉 所有资源统一入口（非常关键）

```sql
CREATE TABLE favorite_items (
  id TEXT PRIMARY KEY,

  type TEXT NOT NULL, -- mod / server / modpack / resourcepack / datapack

  name TEXT NOT NULL,
  icon TEXT,

  source TEXT NOT NULL, -- local / curseforge / modrinth / external
  source_id TEXT,       -- 平台ID（modrinth slug / curseforge id）

  external_url TEXT,

  mc_versions TEXT, -- JSON数组 ["1.20.1", "1.21.1"]
  loaders TEXT,     -- JSON数组 ["fabric", "forge"]

  created_at INTEGER,
  updated_at INTEGER
);
```

# 四、核心查询设计（直接可用）

---

## 1️⃣ MOD筛选（核心功能）

### 条件：

* group = 农业
* loader = fabric
* mc = 1.21.1

```sql
SELECT fi.*
FROM favorite_items fi

JOIN favorite_group_map gm ON fi.id = gm.favorite_id
JOIN groups g ON gm.group_id = g.id

JOIN favorite_versions fv ON fi.id = fv.favorite_id

WHERE fi.type = 'mod'
  AND g.name = '农业'
  AND fv.loader = 'fabric'
  AND fv.mc_version = '1.21.1'
  AND fv.is_available = 1;
```

---

## 2️⃣ 检查分组是否全部支持目标版本（🔥核心功能）

```sql
SELECT COUNT(*) as total,
       SUM(CASE WHEN fv.is_available = 1 THEN 1 ELSE 0 END) as supported
FROM favorite_group_map gm

JOIN favorite_versions fv ON gm.favorite_id = fv.favorite_id

WHERE gm.group_id = ?
  AND fv.mc_version = ?
```

👉 判断：

```ts
if (total === supported) {
  // 全部支持 → 触发弹窗
}
```

---

## 3️⃣ 查询关联资源（下载提示）

```sql
SELECT fi.*
FROM relations r
JOIN favorite_items fi ON r.to_id = fi.id
WHERE r.from_id = ?
```

# 九、你要注意的坑（很重要）

---

## ⚠️ 1. MC版本不是字符串问题

👉 `"1.20"` vs `"1.20.1"`

建议：

```ts
normalizeVersion("1.20") → "1.20.0"
```

---

## ⚠️ 2. loader混乱

统一：

```
fabric / forge / neoforge / quilt
```

---

## ⚠️ 3. 外链不可控

👉 必须允许：

* 手动编辑名称
* 手动指定版本

---

## ⚠️ 4. 数据量问题

如果用户收藏很多：

👉 `favorite_versions` 可能爆炸

解决：

* 只缓存“相关版本”
* 或懒加载
