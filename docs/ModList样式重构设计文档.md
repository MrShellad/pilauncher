# ModList 样式重构设计文档：高密度与可管理性优化

## 1. 背景与目标
随着用户安装的 Mod 数量增加（通常超过 50 个，甚至达到数百个），现有的基于 `OreAssetRow` 的大列表布局在信息获取效率上会显著下降。过大的图标、冗余的垂直间距以及分散的信息分布导致用户需要频繁滚动，且难以在大规模列表中快速定位特定 Mod。

本方案旨在重构 ModList 的视觉与交互设计，实现：
- **高信息密度**：单屏显示的条目数量增加 2-3 倍。
- **低视觉疲劳**：减少干扰线条，优化色彩对比，利用视觉重心引导。
- **大规模管理**：支持批量操作、快速过滤与分组展示。

---

## 2. 核心设计理念

### 2.1 模块化视图模式
提供三种可选的视图模式，满足不同场景下的用户需求：

| 模式 | 适用场景 | 关键特征 |
| :--- | :--- | :--- |
| **标准视图 (Standard)** | 探索与识别 | 64px 图标，双行文本（标题+描述），适合少量 Mod 或初次整理。 |
| **紧凑视图 (Compact)** | 日常快速定位 | 32px 缩略图，单行文本，隐藏描述，适合 50-100 个 Mod。 |
| **数据列表 (Data Grid)** | 深度管理 | 移除图标，多列展示（版本、大小、来源、启用状态），类似文件管理器。 |

### 2.2 视觉降噪
- **去除冗余边框**：放弃每个条目使用独立重边框的设计，转而使用极细的底边分隔线或交替背景色（Zebra Striping）。
- **色彩语义化**：使用侧边状态条（Accent Bar）而非大面积背景色来表示“选中”或“操作中”状态。
- **文本层级优化**：
    - 标题：加粗，深色。
    - 版本号：等宽字体，灰色背景小标签。
    - 文件名/路径：浅灰色，较小字号。

---

## 3. 详细设计方案

### 3.1 紧凑型条目布局 (Compact Row)
紧凑模式下，每个条目的高度从原来的 ~80px 压缩至 **40px-44px**。

- **Leading**: 32x32px 缩略图，若无图标则显示 Mod 类型对应的微型图标（如 Blocks, Bug, Tool）。
- **Center**: 
    - 第一行：`[标题] [版本号 badge] [更新可用 badge]`
    - 第二行（可选/悬浮显示）：`[文件名] | [大小]`
- **Trailing**: 
    - 简单的开关切换器（Switch）。
    - 更多操作按钮（三点图标）在 Hover 时显示，减少常驻干扰。

### 3.2 智能过滤与分组 (Smart Filtering)
针对大量 Mod，必须提供高效的检索手段：
- **常驻快捷过滤器**：
    - `全部` | `已启用` | `已禁用` | `可更新` | `外部/手动添加`
- **自动分组 (Category Grouping)**：
    - 根据 Mod 来源或元数据自动归类为：`基础库 (Libraries)`、`游戏内容 (Content)`、`性能优化 (Performance)`、`未分类`。
    - 每一组提供一个可折叠的 Header。

### 3.3 批量操作 (Batch Operations)
- **多选机制**：
    - 支持键盘 `Shift + 点击` 连续选择。
    - 列表左侧提供隐藏的复选框，Hover 或进入编辑模式时显示。
- **浮动操作栏**：
    - 当选中多个条目时，底部弹出浮动栏，提供：`批量启用`、`批量禁用`、`批量删除`、`移动到分组`。

---

## 4. 交互优化

### 4.1 焦点导航 (Focus Navigation)
- **极速搜索**：在列表聚焦时直接输入字符，自动跳转到首字母匹配的条目。
- **虚拟化列表 (Virtualization)**：
    - 必须实现完全的 DOM 虚拟化（如 `react-window`），确保在 500+ Mod 时依然能保持 60fps 的滚动性能。

### 4.2 悬浮预览 (Hover Inspection)
- 减少点击进入详情的频率。
- 当用户将光标悬停在条目上 500ms 后，在侧边或浮窗显示该 Mod 的完整描述、依赖关系和截图。

  --modlist-bg-hover: rgba(255, 255, 255, 0.04);

.mod-row-compact {
  display: grid;
  grid-template-columns: 32px 1fr auto;
  align-items: center;

  padding: 4px 12px;
  border-bottom: 1px solid var(--modlist-border-color);
