# 游戏启动后手柄监听自动暂停方案设计文档

## 1. 需求背景
当前启动器集成了完整的手柄导航支持（InputDriver）。然而，当玩家启动 Minecraft 游戏后，手柄输入通常由游戏直接接管。如果启动器在后台继续监听手柄并模拟键盘操作，会导致：
- **双重输入**：玩家在游戏中操作手柄时，后台的启动器也会同步移动焦点或触发动作。
- **性能损耗**：后台持续轮询手柄状态会占用不必要的资源。

## 2. 核心思路
通过监听 `useGameLogStore` 中的 `gameState` 状态，当游戏处于 `running`（运行中）或 `launching`（启动中）阶段时，暂时挂起 `InputDriver` 的手柄处理逻辑。

## 3. 调整位置

### 3.1 `src/ui/focus/InputDriver.ts`
在 `InputDriver` 的主循环（`loop` 函数）中增加状态校验。

**逻辑调整示例：**
```typescript
const { gameState } = useGameLogStore.getState();

// 如果游戏正在运行或启动中，跳过手柄逻辑处理
if (gameState === 'running' || gameState === 'launching') {
    currentGamepadActions.clear(); 
}
```

### 3.2 兼容性考虑
- **手动恢复**：当游戏退出（状态回到 `idle` 或 `crashed`）时，`InputDriver` 会自动恢复对手柄的响应。
- **键盘/鼠标不受影响**：通常建议保留键盘鼠标的监听，以便玩家在切回启动器查看日志时仍能操作。

## 4. 实施方案

### 第一步：引入 Store
在 `src/ui/focus/InputDriver.ts` 中引入 `useGameLogStore`。

### 第二步：修改 `loop` 逻辑
在 `useInputDriver` 的 `useEffect` 内部的 `loop` 函数中，获取当前游戏状态并进行截断。

### 第三步：清理状态
当状态切换为非活动时，务必清空 `nativeButtonsRef` 和 `nativeAxesRef`，防止在游戏运行期间残留的按键状态在游戏关闭后突然触发。

## 5. 预期效果
1. 点击启动按钮后，手柄焦点将固定在当前位置，不再随手柄摇杆移动。
2. 游戏运行期间，手柄的所有操作仅对游戏有效，不会误触启动器的 UI 按钮。
3. 游戏关闭并回到启动器界面后，手柄导航立即恢复正常。
