# PiLauncher 性能优化分析报告

本报告针对 PiLauncher 项目当前的架构与实现细节，从后端（Rust/Tauri）与前端（React/Typescript）两个维度进行深度分析，并提出性能优化建议。

---

## 1. 后端优化建议 (Rust / Tauri)

### 1.1 数据库层 (SQLite) - [已实施]
*   **WAL 模式与同步优化**：
    *   **现状**：默认模式下，SQLite 的并发写性能较低，且在大批量写入时会锁定数据库。
    *   **改进**：已启用 `PRAGMA journal_mode=WAL;` 和 `PRAGMA synchronous=NORMAL;`。这允许读取者与写入者并行工作，极大提升了 UI 响应速度。
*   **批量事务处理**：
    *   **现状**：原先 `get_mods` 会为每个 Mod 启动一个独立的并发写入任务，导致严重的锁竞争。
    *   **改进**：已改为批量合并更新，在扫描完成后使用单一事务处理所有缓存写入。

### 1.2 文件系统 I/O
*   **并发目录扫描**：
    *   **现状**：目前部分模块（如 `save_manager.rs`）可能还在使用标准库的 `read_dir`。
    *   **建议**：对于文件数量巨大的目录（如大型整合包的 `mods` 或 `saves` 备份），建议引入 `jwalk` 或 `rayon` 进行多线程并发遍历，减少冷启动时的磁盘等待时间。
*   **哈希校验优化**：
    *   **建议**：在进行文件完整性校验时，确保使用 `std::io::BufReader`。对于超大文件，可以考虑使用内存映射（`memmap2`），但这需权衡内存占用。

### 1.3 资源处理
*   **图标缩略图化**：
    *   **现状**：目前直接从 JAR 中提取原始图标或从网络下载并在前端显示。
    *   **建议**：在后端对提取的 Mod 图标进行缩放处理（例如统一缩放到 64x64），并转换为 WebP 格式。这能显著减少 Tauri 主进程与 webview 之间的 IPC 序列化数据量，提升列表滑动流畅度。

## 2. 前端优化建议 (React / Vite)

### 2.1 列表渲染性能
*   **虚拟化长列表 (Virtualization)**：
    *   **现状**：`ResourceGrid` (下载页) 与 `ModList` (实例详情) 均采用 `.map` 直接渲染所有 DOM。当搜索结果或模组数量超过 100+ 时，DOM 节点过重会导致页面卡顿。
    *   **建议**：引入 `react-window` 或 `react-virtuoso` 仅渲染可视区域内的卡片。
*   **复杂组件 Memoization**：
    *   **建议**：对 `ResourceCard` 和 `ModRow` 使用 `React.memo` 进行包裹，并配合 `useCallback` 传递事件回调，避免父容器更新时无关子项的重绘。

### 2.2 逻辑计算瓶颈
*   **状态查找优化**：
    *   **现状**：在 `ResourceGrid` 中，判断“模组是否已安装”使用的是 `installedMods.some(...)`，复杂度为 $O(N \times M)$。
    *   **建议**：在 `useResourceDownload` hook 中，将 `installedMods` 预处理为一个以 `projectId` 为键的 `Set` 或 `Map`。将查找复杂度从 $O(N)$ 降至 $O(1)$。

### 2.3 状态管理 (Zustand)
*   **选择器 (Selectors) 优化**：
    *   **建议**：确保在组件内使用 Zustand 的精确选择器（如 `useLauncherStore(state => state.selectedInstanceId)`），而不是获取整个 state，避免无关状态变更导致全页刷新。

---

## 3. 总体架构优化

### 3.1 缓存策略
*   **预加载机制**：
    *   **建议**：在用户将鼠标悬停在“下载”侧边栏时，提前在后台触发环境加载和部分分类数据的拉取。

### 3.2 IPC 通信
*   **减少大数据传输**：
    *   **建议**：对于非常长的列表（如数千个文件的清单），避免一次性通过 `invoke` 传递整个 JSON 对象。考虑在后端分页，或使用流式传递。

---

## 优化优先级建议
1.  **[高]** 列表虚拟化（针对下载页）。
2.  **[中]** 状态查找优化（Set 查找）。
3.  **[中]** 图标后端缩放下发。
4.  **[低]** 目录扫描并发化。
