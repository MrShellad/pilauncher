# 📄 Minecraft 启动器内存分配机制设计文档

## 1. 设计目标

### 1.1 核心目标

构建一套**智能、安全、可控、可扩展**的内存分配系统，满足：

* 自动适配不同设备（4GB ~ 64GB+）
* 支持多实例独立配置
* 避免 OOM / 卡顿 / GC 抖动
* 提供用户可控能力（但防止误操作）

---

### 1.2 设计原则

* **安全优先**：绝不因分配策略导致系统卡死
* **自动优先**：默认无需用户干预
* **用户可控**：允许高级用户覆盖
* **动态适配**：根据当前系统状态调整

---

## 2. 核心概念定义

### 2.1 内存参数

| 参数            | 含义            |
| ------------- | ------------- |
| total_ram     | 系统总内存         |
| available_ram | 当前可用内存        |
| allocated_ram | 分配给 MC 的内存    |
| min_ram       | JVM 最小内存（Xms） |
| max_ram       | JVM 最大内存（Xmx） |

---

### 2.2 分配模型

```text
最终分配 = min(推荐值, 安全值)
```

* 推荐值：基于总内存
* 安全值：基于当前可用内存

---

## 3. 内存分配策略

## 3.1 推荐值计算（基于总内存）

```text
recommended = total_ram × 0.6
```

### 修正规则（按实例类型）

| 类型    | 修正     |
| ----- | ------ |
| 原版    | -1GB   |
| 轻量Mod | 基准     |
| 中型Mod | +1~2GB |
| 重型Mod | +3~6GB |

---

## 3.2 安全值计算（基于可用内存）

```text
safe_limit = available_ram × 0.7
```

### 说明：

必须预留内存给：

* 操作系统
* 后台程序
* JVM GC波动

---

## 3.3 最终分配

```text
final_memory = min(recommended, safe_limit)
```

---

## 4. 分配限制机制（强制）

### 4.1 硬限制（Hard Limit）

```text
max ≤ total_ram × 0.8
max ≤ available_ram × 0.8
```

---

## 5. 用户控制机制

### 5.1 模式设计

| 模式       | 描述     |
| -------- | ------ |
| 自动模式（默认） | 系统计算   |
| 手动模式     | 用户指定   |
| 强制模式     | 忽略安全限制 |

---

### 5.2 最终优先级

```text
用户设置 > 自动推荐 > 安全限制
```

最终仍需：

```text
final = min(用户值, safe_limit)
```

（除非开启“强制模式”）

---

## 6.3 按内存规模优化

| 内存     | 策略           |
| ------ | ------------ |
| <4GB   | 小堆优化         |
| 4~12GB | 标准G1GC       |
| >12GB  | 调整RegionSize |

---

## 7. 启动前检测机制

### 7.1 检测项

* 当前可用内存
* 系统负载
* 是否低于阈值
